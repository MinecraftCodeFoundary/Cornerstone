# Cornerstone 开发里程碑

Cornerstone 是一个面向 NeoForge 服务器的基础设施 Mod，定位类似 Essentials：提供传送、社交、管理、运维与经济等通用能力，并为其他玩法 Mod 提供可靠的服务接口。

## 开发原则

- 服务端是权限、传送、处罚、经济与持久化数据的唯一权威。
- 以玩家 UUID 和稳定资源标识保存数据，并为数据版本升级预留迁移路径。
- 核心本体保持轻量：领地、反作弊、商店、拍卖、任务和 RPG 系统通过独立 Mod 或 API 扩展实现。
- 高风险命令必须有细粒度权限、冷却、可配置规则与审计日志。
- 完整支持模组服环境：多维度、模组物品、数据组件、容器和传送机制。

## M0：项目基座（`0.0.1`）

建立可长期维护的 NeoForge 工程。

- 规划包结构：`core`、`command`、`config`、`data`、`api`、`integration`。
- 建立服务端配置加载、校验与重载机制。
- 建立中英文语言文件、统一消息格式和结构化日志。
- 清理模板代码与调试输出，配置构建和基础测试流程。

**验收标准：** 专用服务端可稳定启动和关闭；配置与语言文件生效；Gradle 完整构建通过。

## M1：核心服务层（`0.1.0`，已完成）

构建所有功能共用的底层能力。

- 玩家数据中心：通用资料、最后在线时间、最后位置和模块数据。
- 数据存储抽象：版本字段、原子写入、异常恢复和迁移接口。
- 权限服务：以 LuckPerms `5.4+` 为必需前置，统一使用权限节点授权。
- 命令框架：统一处理权限、参数校验、冷却、消息和执行结果。
- 审计服务：记录执行者、目标、时间、维度、参数摘要和结果。
- 提供稳定的 Java API 与事件，不允许外部 Mod 直接操作内部存档。

**验收标准：** 新模块不需要重复实现权限、保存与日志；重启后数据不丢失；非法参数不会导致服务端异常。

**完成记录（2026-07-15）：**

- [x] 以 UUID 为主键的玩家资料中心，保存最后在线时间、最后位置与受校验的模块数据。
- [x] 带格式版本、CRC32 校验、原子替换、`.bak` 恢复和迁移接口的持久化存储。
- [x] LuckPerms `5.4+` 必需前置与缓存权限数据授权；原版 OP 等级 4 默认拥有全部 Cornerstone 权限。
- [x] 统一命令框架；`/cornerstone status`、`/cornerstone save` 已接入权限、执行结果和审计。
- [x] 记录执行者、目标、时间、维度、参数摘要和结果的审计服务。
- [x] `CornerstoneApi`、玩家资料变更事件、中英文语言文件和 M1 使用文档。
- [x] `./gradlew.bat build --no-daemon` 完整构建通过。

## M2：传送 MVP（`0.2.0`，已完成）

优先实现玩家最常用的服务。

- `/spawn`、`/setspawn`：全局或按维度出生点。
- `/home`、`/sethome`、`/delhome`：可配置数量、名称和维度规则。
- `/tpa`、`/tpaccept`、`/tpdeny`、`/tpacancel`：请求、超时和取消状态。
- `/back`：默认记录死亡位置，可配置是否记录传送前位置。
- 传送落点安全检查：碰撞、危险方块、区块状态和维度限制。
- 统一的延迟、移动取消、战斗限制、冷却与传送费用接口。

**验收标准：** 可稳定跨原版与模组维度传送；家与出生点跨重启保留；不会把玩家传送至明显不可存活的位置。

**完成记录（2026-07-15）：**

- [x] `/spawn`、`/setspawn [global|dimension]`：按当前维度优先、全局出生点回退，均可跨重启保存。
- [x] `/home [name]`、`/sethome [name]`、`/delhome <name>`：名称校验、数量限制和跨重启保存。
- [x] `/tpa <player>`、`/tpaccept`、`/tpdeny`、`/tpacancel`：单目标待处理请求、超时、拒绝和取消反馈。
- [x] `/back`：默认保存死亡位置；可配置为同时记录成功传送前的位置。
- [x] 跨维度目标解析、区块加载、世界边界、构建高度、碰撞、地面和危险方块安全校验。
- [x] 统一延迟、移动取消、冷却、维度黑名单；提供 `TeleportRestriction`（战斗/保护）和 `TeleportCostProvider`（经济）扩展接口。
- [x] `./gradlew.bat build --no-daemon` 完整构建通过。

## M3：地标与随机传送（`0.3.0`，已完成）

完善地图移动能力。

- `/warp`、`/setwarp`、`/delwarp`：公开地标、权限地标和管理员地标。
- `/rtp`：按维度配置范围、最小距离、尝试次数和冷却。
- 支持世界边界、维度黑名单、危险落点与生物群系过滤。
- 为领地或保护 Mod 提供落点校验扩展点。

**验收标准：** 随机传送有次数上限与明确失败反馈；各地标的权限和费用可独立配置。

**完成记录（2026-07-15）：**

- [x] `/warp <name>`、`/setwarp <name> [public|permission|admin]`、`/delwarp <name>`，并以存档 v2 保存地标和访问策略。
- [x] 公开、专属权限和管理员地标；专属节点为 `cornerstone.warp.<name>`，管理员节点为 `cornerstone.warp.admin.<name>`。
- [x] `/rtp` 按维度规则执行半径、最小距离、尝试上限与独立冷却；失败时返回明确次数反馈。
- [x] RTP 校验世界边界、维度黑名单、区块、危险落点和可配置生物群系黑名单，复用传送安全检查。
- [x] `addLandingValidator(...)` 为领地/保护 Mod 提供落点拒绝扩展点；地标费用通过 `teleport.warpCosts` 与 `TeleportContext.configuredCost()` 独立传给经济适配器。
- [x] M2 存档自动迁移至包含地标的 v2 格式；`./gradlew.bat build --no-daemon` 完整构建通过。

## M7：经济 API（`0.7.0`，已完成）

提供可被其他玩法 Mod 复用的经济基础。

- 账户、余额查询和 `/pay`。
- 事务模型：唯一事务 ID、幂等处理、转出/转入记录和失败回滚。
- 余额变动事件与 Java API。
- 可选的经济流水查询和数据导出。

Cornerstone 本体不实现服务器商店、拍卖行或复杂货币玩法。

**验收标准：** 重复请求不会重复扣款；每笔交易可追溯；外部 Mod 无需接触内部存档即可调用经济服务。

**完成记录（2026-07-15）：**

- [x] UUID 账户、整数最小单位余额、可配置开户余额与 `/balance`、`/pay`。
- [x] 转账要求唯一事务 UUID；同一 ID 重放原结果，不会重复扣款；复用不同参数的 ID 会被拒绝。
- [x] 原子持久化前后实施借方/贷方同步更新；写入失败时在内存中回滚余额和事务记录。
- [x] `economy.dat` 保存可追踪流水；提供 `BalanceChangedEvent`、事务查询、玩家历史和 CSV 导出。
- [x] `/economy history <player>`、`/economy export` 使用独立权限并纳入审计。
- [x] `./gradlew.bat build --no-daemon` 完整构建通过。

## 发布门槛

每个版本发布前必须满足：

- Gradle 完整构建和专用服务端启动测试通过。
- 新增持久化数据具有版本、保存、加载和迁移测试。
- 新增命令覆盖权限、非法参数、断线和重启路径。
- 新增传送、经济或管理行为均写入审计记录。
- 涉及模组物品、维度和容器的功能均经过实际整合服验证。
- 配置项、权限节点、数据格式与玩家可见行为同步更新文档。

## 范围控制

在 `1.0.0` 前不优先开发领地、反作弊、商店、拍卖、任务、RPG 属性、菜单 GUI 或大型活动玩法。它们应作为独立 Mod，或通过 Cornerstone API 实现，以保持核心模块稳定、兼容且便于维护。
